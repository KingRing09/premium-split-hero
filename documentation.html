6. Performance Tips

Image Optimization

  • Compress images before uploading (use TinyPNG or Squoosh)
  • Use WebP format when possible for 25-35% smaller file sizes
  • Keep desktop images under 300KB each
  • Keep mobile images under 150KB each

Built-in Optimizations

  • Lazy Loading: Mobile images load only when needed
  • Fetch Priority: Desktop images are prioritized for faster LCP
  • Responsive Images: Shopify automatically serves optimized sizes
  • Minimal JavaScript: ~2KB minified, vanilla JS (no jQuery)
  • CSS Variables: Efficient styling with no duplicate code
üìä Performance: This section is optimized to score 90+ on Google PageSpeed Insights when images are properly compressed.

7. Troubleshooting

Common Issues & Solutions

‚ùì Images not showing

Solution: Make sure you've uploaded images in the Theme Editor for each column block. Placeholder images appear when no image is selected.

‚ùì Animations not working

Solution: Check if your theme has CSS that might conflict. The section uses the prefix hero- for all classes to minimize conflicts. If issues persist, check for !important rules in your theme CSS.

‚ùì Mobile cards not expanding

Solution: Ensure JavaScript is enabled. Check your browser console for errors. Some themes may have conflicting JavaScript.

‚ùì Center content not fading on hover

Solution: This is intentional behavior - the center content fades when you hover over any column. It will reappear when you move your mouse away.

‚ùì Fonts look different

Solution: The section loads Poppins and Inter fonts from Google Fonts. If your theme blocks external fonts, you can change the font-family in the CSS variables.

‚ùì Section appears too tall/short

Solution: Adjust the "Section Height" setting in the Theme Editor (range: 50vh - 100vh).
